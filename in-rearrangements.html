 (require "in-rearrangements.rkt")

procedure

(in-rearrangements lst [eq]) → (Sequenceof list?)

  lst : list?
  eq : (any/c any/c . -> . any/c) = equal?
Returns a sequence of all distinct rearrangements of lst. A rearrangement of a list is a list of the same elements but in arbitrary order, the initial order in lst included. eq must be an equivalence relation for the elements of lst. What eq does with other arguments than elements of lst is irrelevant. Two rearrangements are distinct if and only if eq returns #f for at least one pair of corresponding elements. Examples:

> (require racket "in-rearrangements.rkt")
> (define (example lst (eq equal?))
   (for ((x (in-rearrangements lst eq)))
    (printf "~s~n" x)))
> (example '())

()

> (example '(a))

(a)

(a c b)

(b c a)

(b a c)

(c a b)

(c b a)

> (example '(a a b))

predicate

(equivalence? eq lst) → boolean?

  eq : (-> any/c any/c)
  lst : list?
Procedure in-rearrangements does not check eq to be an equivalence relation for the elements of lst. Predicate equivalence? can be used to check whether or not a given procedure eq can be used as an equivalence relation for the elements of a given lst.

Examples:

(equivalence? = (range 100)) → #t
(equivalence? >= (range 100)) → #f
(equivalence? (λ (x y) (eq? (even? x) (even? y))) (range 100)) → #t
